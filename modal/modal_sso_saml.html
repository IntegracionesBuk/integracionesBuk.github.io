Configurar SSO SAML
 

Debes hacer click para acceder a los ajustes de la plataforma.

Parámetros Generales

Aqui encontraremos todas las configuraciones que nos entrga la plataforma.

Debes ingresar la palabra Saml para encontrar la configuración.

Habilitar SAML Activa el servicio de autenticación SAML
Dominios a autenticar vía SAML Correos que cuenten con el dominio definido en estos parámetros serán consultados por Buk al proveedor de autenticación.
Issuer/Audience URL (opcional) Es un campo de texto libre personalizado. Suelen ser similares a contraseñas o URL’s.
Authentication Context (opcional) Método extra de autenticación de SAML.
URL del IdP para SSO Esta URL es la que Buk utilizará para ir a consultar hacía el proveedor de autenticación del cliente sobre la existencia del usuario.
URL del IdP para SLO Se utiliza solo para que cuando el usuario cierre sesión en BUK, se le cierre además la sesión en el proveedor SAML.
Certificado de IdP Es la información con la cual el proveedor de autenticación puede validar que la transacción es válida.
Hash del certificado de IdP huella digital del certificado del proveedor de identidad. No es necesario si se tiene el certificado IdP.
Url Idp Logout forzada URL de destino para SIngle Log Out del proveedor de identidad. Dejar vacío permite que solo se cierre sesión en buk.
Algoritmo del hash del certificado Algoritmo para el hash de certificado, generalmente es utilizado el sha256

Luego de configurar cada uno de los campos solicitados, daremos click en el botón guardar.
